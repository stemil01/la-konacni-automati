# *Лексичка анализа и примене*: семинарски радови са коначним аутоматима

Ово је костур за семинарске радове који имају за циљ формирање коначних аутомата на основу регуларних израза.
Цела суштина костура је да трансформише задати регуларни израз у репрезентацију погодну за обраду у неком програмском језику, што је у овом случају *Python*.

## Пример употребе

Потребно је укључити функцију `parse_regex`:
```python
from parser.RegexParser import parse_regex
```
која као аргумент прима ниску која представља валидан регуларни израз.
Повратна вредност функције је дрволика репрезентација регуларног израза.

Конкретно, регуларни изрази могу да садрже наредно:
1. ASCII карактер, нпр. `a`
2. конкатенацију, нпр. `ab`
3. унију, нпр. `a|b`
4. Клинијево затворење, нпр. `a*`

За сваки ће се формирати одговарајућа *Python* торка (tuple), и то:
1. ASCII карактер ће бити `('char', karakter)`
2. конкатенација ће бити `('concat', levi_podizraz, desni_podizraz)`
3. унија ће бити `('union', levi_podizraz, desni_podizraz)`
4. Клинијево затворење ће бити `('star', podizraz)`

На пример, регуларни израз `(a|b)*abb` ће бити представњен као:
```
('concat',
 ('concat',
  ('concat', ('star', ('union', ('char', 'a'), ('char', 'b'))), ('char', 'a')),
  ('char', 'b')),
 ('char', 'b'))
```
